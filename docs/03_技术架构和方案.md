## 4. 核心数据模型

### 4.1 CandidateProfile

表示用户画像。

字段包括：

- 工作年限
- 学历背景
- 有无产品经验
- 过往产品类型
- 熟悉的技术
- 擅长的技能
- 目标职位

用途：

- 参与能力分析
- 参与面试重点生成
- 参与求职建议生成

### 4.2 JobPosting

表示拆分后的单个岗位。

字段包括：

- 岗位标题
- 原文
- 薪资信息
- 关键词
- 能力标签
- 解析状态

### 4.3 AnalysisResult

表示一次 JD 分析的完整结果。

字段包括：

- 岗位数量
- 平均薪资
- Top关键词
- 候选人核心能力要求
- 能力分析
- 面试重点
- 岗位洞察
- 求职建议
- 岗位明细数据

## 6. 岗位拆分方案

早期拆分规则会误把 `【岗位职责】`、`【任职要求】`、`【加分项】` 当成新岗位，导致单条 JD 被切成多条。

当前方案：

- 优先识别多个明确岗位标题。
- 当文本中出现多个 `岗位名称：`、`职位名称：`、`岗位：`、`职位：` 时，才按多岗位拆分。
- `【岗位职责】`、`【任职要求】`、`【加分项】` 被视为同一岗位内部结构。
- 如果无法可靠拆分，则整段作为 1 个岗位处理。

## 7. 薪资解析方案

支持格式：

- `20-40K`
- `20k-40k`
- `25-35K·14薪`
- `面议`

计算方式：

- 区间薪资取中位数。
- 带 `14薪` 等信息时保留展示标签，但平均月薪仍按月薪区间中位数估算。
- 面议或无法识别薪资不参与平均薪资计算。

## 8. 关键词提取方案

使用本地关键词规则字典。

每个关键词包含：

- 标准关键词
- 同义词 / 别名
- 类别
- 对应能力域

示例：

```ts
RAG -> 检索增强、知识库、向量库、召回、检索
Prompt -> prompt、提示词、提示工程、提示词工程
Agent -> agent、智能体、工具调用、function calling
模型评测 -> 评测、效果评估、benchmark、准确率、幻觉、满意度
```

## 9. 能力映射方案

关键词会映射到能力域：

```text
RAG -> RAG / 知识库
Prompt -> Prompt 工程
Agent -> Agent 产品设计
LLM / 问答 / 多模态 -> AI 应用设计
评测 / 准确率 / 幻觉 -> 模型评测
数据分析 / A/B 测试 -> 数据分析
需求分析 / PRD / 原型 -> 产品方法论
跨部门 / 协同 / 落地 -> 项目推进
```

能力要求模块根据能力出现次数和排序生成星级强度。

## 10. 能力分析方案

能力分析结合两类输入：

- JD 能力要求
- 用户画像

核心判断：

- 优势能力：画像中已有直接证据，且 JD 中有相关要求。
- 可迁移能力：画像没有完全覆盖，但可从产品经验、技术熟悉度或技能迁移。
- 重点补齐：JD 中重要，但用户画像证据不足。

简化评分逻辑：

```text
能力差距优先级 =
JD 提及强度 45%
+ AI 岗位关键度 30%
+ 用户缺口程度 25%
```

画像匹配信号来源：

- 熟悉技术
- 擅长技能
- 过往产品类型
- 有无产品经验
- 工作年限

## 11. 面试重点方案

面试重点基于能力分析结果生成。

类型包括：

- 必备问题
- 案例包装
- 短板补救
- 临时补课

生成逻辑：

- 从高频能力中生成高概率面试问题。
- 从用户过往产品类型中生成经验迁移表达。
- 从重点补齐能力中生成短板补救话术。
- 从 AI 高频能力中生成临时补课清单。

## 13. Chrome 扩展方案

扩展类型：

- Chrome Manifest V3
- Content Script

功能：

- 在 BOSS 页面右下角注入悬浮面板。
- 从当前页面 DOM 中提取岗位标题、薪资和岗位详情。
- 支持复制当前岗位。
- 支持加入暂存。
- 支持导出暂存。
- 支持清空暂存。

安全边界：

- 不请求 BOSS 接口。
- 不绕过反爬。
- 不自动批量爬取。
- 只读取用户当前打开页面的可见文本。

## 15. 当前技术边界

- 不保存用户输入。
- 不支持用户登录。
- 不调用 AI API。
- 不支持历史记录。
- 不支持服务端数据分析。
- 国内移动网络访问 Vercel 可能不稳定。
