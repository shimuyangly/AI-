# AI求职助手 MVP PRD

## 1. 产品概述

### 1.1 产品名称

AI求职助手

### 1.2 产品定位

面向 AI 产品经理求职者的岗位 JD 分析助手，帮助传统产品经理通过分析真实招聘岗位，理解 AI PM 岗位能力要求，并获得更贴合个人画像的能力差距分析和求职建议。

### 1.3 产品目标

- 降低用户人工整理多个岗位 JD 的时间成本。
- 帮助用户识别 AI 产品经理岗位中的高频关键词和能力要求。
- 结合用户画像，给出能力差距和面试准备建议。
- 为传统产品经理转型 AI PM 提供结构化求职参考。

## 3. 用户场景

### 3.1 场景一：批量分析岗位 JD

用户从 BOSS 直聘复制多个 AI 产品经理岗位正文，粘贴到系统中，希望快速知道这些岗位共同要求什么能力。

### 3.2 场景二：判断自己与岗位要求的差距

用户已经有传统产品经验，但不知道自己距离 AI PM 岗位要求差在哪里，希望系统结合个人画像给出能力差距判断。

### 3.3 场景三：准备面试

用户准备投递或面试 AI 产品经理岗位，希望基于 JD 高频要求生成面试准备重点。

### 3.4 场景四：降低岗位信息复制成本

用户需要从 BOSS 直聘复制大量岗位信息，希望通过浏览器扩展减少重复整理工作。

## 4. 功能需求

### 4.1 岗位详情输入

功能说明：

- 支持粘贴岗位正文。
- 不支持直接粘贴岗位 URL。
- 支持一次粘贴多个岗位。
- 最多支持 50 个岗位。
- 字数不做限制。

输入提示：

1. 请粘贴岗位名称、薪资、岗位介绍等内容。
2. 一次最多支持粘贴 50 个岗位信息。
3. 岗位与岗位之间需保留 1 行空白间距，用于区隔不同职位信息。

异常状态：

- 输入为空时，点击分析弹窗提示：请先粘贴岗位详情！
- 超过 50 个岗位时，弹窗提示用户删减后再分析。

### 4.2 数据概览

展示内容：

- 岗位数量
- 平均薪资

计算逻辑：

- 岗位数量来自拆分后的岗位数。
- 平均薪资按可解析薪资区间的中位数计算。
- 面议或无法识别薪资不参与平均薪资计算。

### 4.3 Top关键词

展示内容：

- 高频关键词
- 提及次数
- 条形图占比

关键词范围：

- RAG
- Prompt
- Agent
- LLM
- 问答
- 模型评测
- 数据分析
- A/B 测试
- 需求分析
- 项目推进
- 商业化
- 增长
- B 端
- C 端
- 多模态
- 工作流

### 4.4 候选人核心能力要求

展示内容：

- 能力名称
- 能力说明
- 星级强度

能力域包括：

- RAG / 知识库
- Prompt 工程
- Agent 产品设计
- AI 应用设计
- 模型评测
- 数据分析
- 产品方法论
- 项目推进
- 商业化理解
- 增长产品
- B 端产品
- C 端产品

### 4.5 能力分析

展示内容：

- 能力名称
- 状态标签
- 优先级
- 分析原因
- 建议动作
- JD 证据

状态标签：

- 优势能力
- 可迁移能力
- 重点补齐

优先级：

- 高
- 中
- 低

核心逻辑：

- 根据 JD 能力提及强度、AI 岗位关键度、用户画像匹配程度综合计算。
- 判断用户当前能力与岗位要求之间的关系。

### 4.6 面试重点

展示内容：

- 面试准备标题
- 类型标签
- 优先级
- 准备说明
- 回答要点

类型标签：

- 必备问题
- 案例包装
- 短板补救
- 临时补课

核心逻辑：

- 根据 JD 高频能力和能力差距生成面试准备内容。
- 结合用户过往产品类型和擅长技能，生成可迁移案例表达。

### 4.7 岗位洞察

展示内容：

- 岗位共性总结
- 高频能力趋势
- AI 专项能力和传统产品能力占比判断
- 薪资解析说明

### 4.8 求职建议

展示内容：

- 简历开头建议
- 作品集或 Demo 准备方向
- 面试准备建议
- 过往经历迁移建议

### 4.9 Chrome 扩展

功能说明：

- 在 BOSS 直聘页面展示悬浮面板。
- 支持复制当前岗位。
- 支持加入暂存。
- 支持导出暂存。
- 支持清空暂存。
- 最多暂存 50 个岗位。

边界：

- 不调用 BOSS API。
- 不绕过反爬。
- 只读取当前页面可见文本。

## 5. 信息架构

页面结构：

1. 顶部产品标题和用户画像卡片
2. 岗位正文输入区
3. 数据概览
4. 模块导航 tab
5. Top关键词
6. 候选人核心能力要求
7. 能力分析
8. 面试重点
9. 岗位洞察
10. 求职建议

## 6. 交互设计

- 首屏直接展示可用工具，不做营销落地页。
- 分析后展示卡片式看板。
- 导航 tab 支持点击锚定对应模块。
- 页面上滑时 tab 吸顶。
- 滚动到不同模块时自动高亮对应 tab。
- 卡片内容超出高度时支持内部滚动。
- 移动端采用单列堆叠。

## 7. 非功能需求

- 无需登录。
- 不保存用户输入。
- 刷新页面后数据丢失。
- 无需 API key。
- 本地浏览器完成分析。
- 支持桌面端和移动端基础响应式访问。
